What Black Void Linux is

Built on Void Linux

Black Void Linux starts from the Void Linux base system and package ecosystem. It keeps the same core tools like runit and xbps and follows the rolling release model.

Opinionated styling and defaults

The focus is on a very dark palette near 16,16,16, minimal background services and a layout that keeps the desktop quiet so your work stands out.

No project level telemetry added

Black Void Linux images do not add extra analytics or tracking services. What runs on your system is visible and under your control.

Separate and unofficial

This is an independent project built on the work of the Void Linux community. It is not an official Void Linux flavor and is not endorsed by the Void project.

Under the hood

Init system

Uses runit with simple service directories and predictable boot behavior. Services are regular scripts that are easy to inspect and control.

Package manager

Uses xbps with the Void package tree. For example: xbps-install -S pkgname, xbps-query -Rs pattern and xbps-install -Su for full upgrades.

See the docs for more detailed xbps usage.

Kernel

Based on the monolithic Linux kernel with configuration tuned for a responsive desktop and workstation experience.

Package tree

Packages are built using the void-packages build system so you can reuse existing knowledge and documentation.

Core features

Monochrome focused UI

Consistent grayscale branding and dark system theming designed for focus and reduced visual noise.

Stable rolling base

Follows a rolling release model built on the Void base with an emphasis on predictable and repeatable updates.

Fast boot and login

Lightweight init, trimmed services and reasonable defaults keep startup and login times short.

Minimal images

Images ship with a small default set of packages so you can grow the system to match your workflow.

Available editions

Desktop

Graphical desktop image with a dark theme and console tools preinstalled. Aimed at everyday work and general purpose use.

Includes a preconfigured desktop environment with a monochrome theme. The docs list the current default environment.

Minimal

Bare bones system with networking and tooling needed to bootstrap your own environment for containers, labs or tiny VMs.

Base

Adds common CLI tools and utilities on top of the minimal base. A solid starting point for servers and workstations.

Server

Headless build with conservative defaults, suitable for home labs and remote machines where you want a small and quiet system.

Gaming

Graphics focused image with a tuned Mesa and Vulkan stack. Intended for machines that spend most of their time in games.
