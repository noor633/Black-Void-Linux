Installing Black Void Linux

This guide follows the Void Linux installation manual but is rewritten for Black Void images and defaults. Everything is explained so beginners can follow along easily, even if you’ve never installed Linux before.

1. Download and Prepare Install Media

Go to the download section and get the latest Black Void ISO image. Write it to a USB stick using a tool you are comfortable with:

  • Windows: Use Rufus or Balena Etcher.
  • macOS: Use Balena Etcher or dd from the terminal.
  • Linux: Use dd, Etcher, or gnome-disks.
Tip: Verify the SHA256 checksum to ensure the image isn’t corrupted.

2. Boot from USB & Partition

Insert the USB and boot your computer. Select the correct boot option (UEFI or BIOS). Once in the live environment:

  • Use GParted or the installer’s partition tool to create partitions: / (root), optionally /home, and swap (if needed).
  • Tip: For beginners, choose Ext4 filesystem for simplicity.
  • Remember your partitioning scheme—it will be needed in the next steps.
UEFI systems should have a small EFI partition (~512MB, FAT32) with the "boot" flag.

3. Run the Installer

Black Void uses the Void-style installer. Steps include:

  • Select your target disk and partitions.
  • Set the root password and optionally create a user account.
  • Choose system configuration options (keyboard layout, hostname, network).
  • Install the base system and optional packages (desktop, server, minimal).
The installer shows hints for each step—read them carefully. If unsure, stick with defaults.

4. Post-Install Setup

After the first boot:

  • Log in with your user account or root.
  • Update your system with sudo xbps-install -Su.
  • Install additional software as needed using XBPS or GUI package managers.
  • Configure your desktop environment, network, and any peripherals.
Tip: Keep a live USB handy for recovery or reinstall if needed.

5. Optional Tools & Tips

  • Use Balena Etcher or Rufus for reliable USB writing.
  • Always verify ISO checksums to prevent corrupted installs.
  • If new to Linux, consider installing a GUI environment for easier usage.
  • Check the System documentation after installation to configure runit services, networking, and users.
